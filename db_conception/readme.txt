Описание стуктуры базы данных:

Таблица Users: покупатели интеренет магазина. 
id_user - Primary Key (суррогатный ключ)
login - логин пользователя. Unique. Not Null
password - пароль пользователя. Хэширование MD5. Not Null
email - эл. почта. Unique. Not Null. Уникальное для однозначной обратной связи с пользователем
id_role - роль пользователя(администратор, покупатель). Foreign Key (внешний ключ, таблица Roles)
Можно дополнить полями ФИО, телефон, адрес.

Таблица Roles: роли пользователей.
id_role - Primary Key (суррогатный ключ)
role - роль пользователя (администратор, покупатель). Unique. Not Null

Таблица Goods:
Товары интернет-магазина. 
id_goods - Primary Key (суррогатный ключ)
name - наименование товара. Уникальность не обязательна. Not Null
price - стоимость. Если стоимости нет (еще не определена), товар не доступен для помещения в корзину, 
но доступен для просмотра
details - описание товара
id_good_status - статус товара (в наличии, ожидается прибытие). Foreign Key (внешний ключ, таблица Good_Status)

Таблица Goods_status:
Статус товара - в наличии, ожидается прибытие. Товары, ожидающие прибытие могут быть в корзине, могут быть в заказе, 
но статус заказа не может быть изменен на завершенный, если товар еще не изменил статус на "в наличии".
id_good_status - Primary Key (суррогатный ключ)
status - статус товара (в наличии, ожидается прибытие). Unique. Not Null

Таблица Basket:
Корзина покупателя. Выбор покупателем определенных товаров определенного количества для дальнейшего формления заказа.
id_basket - Primary Key (суррогатный ключ)
id_user - id пользователя, сделавшего заказ. Foreign Key (внешний ключ, таблица Users)
id_good - id выбранного товара. Foreign Key (внешний ключ, таблица Goods)
amount - количество выбранного товара. Not Null.
price - суммарная стоимость выбранного товара. price = amount * goods.price

Таблица Orders - выбранные пользователем из корзины товары (все или только часть) для оформления заказа.
id_order - Primary Key (суррогатный ключ)
id_basket - Foreign Key (внешний ключ, таблица Basket)
id_user - Foreign Key (внешний ключ, таблица Basket). пользователь выбравший товар и сделавший заказ
date - дата оформления заказа. Not Null. 
id_order_status - статус заказа (ожидает оплаты, оплачен, завершен). Foreign Key (внешний ключ, таблица ORDER_STATUS)

Таблица Order_Status
Статус заказа - оплачен, ожидает оплаты, завершен. Заказ может быть заверешн, если товар есть в наличии 
и произведена оплата. 
id_order_status - Primary Key (суррогатный ключ)
status - статус заказа (ожидает оплаты, оплачен, завершен). Not Null. Unique
