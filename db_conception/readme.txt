Описание стуктуры базы данных:

Таблица User: покупатели интеренет магазина. login обязательно д.б. уникальным, email тоже, 
для однозначной идентификации и обратной связи с покупателем
Можно дополнить полями ФИО, телефон, адрес.

Таблица Goods:
Товары интернет - магазина. Может иметь несколько статусов

Таблица Goods_status:
Статус товара - в наличии, ожидается прибытие. Товары, ожидающие прибытие могут быть в корзине, могут быть в заказе, 
но статус заказа не может быть изменен на завершенный, если товар еще не изменил статус на "в наличии"

Таблица Basket:
Выбор покупателем определенных товаров определенного количества для дальнейшего формления заказа

Таблица Order - выбранные пользователем из корзины товары (все или только часть) для выполнения оплаты

Таблица Order_Status
Статус заказа - оплачен, ожидает оплаты, завершен. Заказ заверешн, если товар есть в наличии и произведена оплата.
